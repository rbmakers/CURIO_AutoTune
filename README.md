## 0. 總覽：2×2 版本矩陣

四份程式可以用兩個正交的維度來理解：「調校演算法世代」(L1/L2) 與「控制架構」(Angle單環/Dual雙環串級)。

| | **單環 (Angle)** | **雙環串級 (Dual / Cascade)** |
|---|---|---|
| **L1 世代**<br>(Ziegler–Nichols No-Overshoot) | `CURIO_L1_Angle_AutoTune.ino` | `CURIO_L1_Dual_AutoTune.ino` |
| **L2 世代**<br>(Tyreus–Luyben) | `CURIO_L2_Angle_AutoTune.ino` | `CURIO_L2_Dual_AutoTune.ino` |

四份程式共用同一套繼電器回授 (Relay Feedback) 量測機制與同一套硬體/安全防護設計，差異集中在「調校演算法」與「控制架構」這兩個維度上。本文件第 1 節先深入比較 L1 與 L2 兩種調校演算法的本質差異與優劣，第 2 節再逐一介紹四份程式的實作內容。

## 1. L1 vs L2：兩種繼電器回授調校演算法比較

### 1.1 共同基礎：繼電器回授量到的是什麼

四份程式都採用 Åström–Hägglund 繼電器回授法 (1984)：暫時用一個 `±d` 的繼電器取代 PID，誘發系統產生穩定的「極限振盪」(limit cycle)，再由實測的振盪振幅 `a` 與週期 `Pu` 反推出系統在 180° 相位處的「極限增益」：

```
Ku ≈ 4d / (π·a)        極限增益
Pu                     極限振盪週期 (對應極限角頻率 ωu = 2π/Pu)
```

**L1 與 L2 量測 Ku、Pu 的方式完全相同**，差異只在於量到 Ku、Pu 之後，要用哪一套公式換算成最終的 Kp、Ki、Kd。這正是「L1」與「L2」這兩個世代名稱真正代表的意義——兩種不同的**換算規則 (Tuning Rule)**，與控制架構(單環/雙環)無關。

### 1.2 L1 採用：Ziegler–Nichols「No Overshoot」規則

L1 系列的 `enum TuningRule` 完整保留了 Ziegler–Nichols 極限增益法的四個經典規則(`ZN_CLASSIC_PID`、`ZN_PESSEN_INTEGRAL`、`ZN_SOME_OVERSHOOT`、`ZN_NO_OVERSHOOT`)，並選用其中最保守的 **No Overshoot**：

```
Kp = 0.20 · Ku
Ti = 0.50 · Pu        (Ki = Kp / Ti)
Td = 0.333 · Pu       (Kd = Kp · Td)
```

這是 1942 年 Ziegler 與 Nichols 提出的原始極限增益法之後，業界針對「希望完全不過衝」的場合所發展出的保守變體——四個規則裡 Kp 係數最小(0.2 倍 Ku)，目的是用較小的比例增益換取響應過程中幾乎不超過目標值。

### 1.3 L2 採用：Tyreus–Luyben 規則

L2 系列換用 **Tyreus–Luyben (1992)** 規則：

```
Kp = Ku / 3.2            (≈ 0.3125 · Ku)
Ti = 2.20 · Pu
Td = Pu / 6.5            (≈ 0.1538 · Pu)
```

> **歷史脈絡**：Tyreus–Luyben 規則最初是針對化工程序(如蒸餾塔)的 **PI** 控制器設計，目標是在程序工業常見的大時間常數系統中提供比 Ziegler-Nichols 更高的穩定裕度。原始論文只給出 Kp、Ti，沒有微分項；程式中額外加上的 `Td = Pu/6.5` 是業界常見、但非原始論文的 PID 延伸經驗值。這點在比較兩者「微分項」優劣時要留意其來源並非同一層級的嚴謹度。

### 1.4 數學比較：相同 Ku、Pu 下的增益差異

把兩條公式直接相除，可以得到一個跟 Ku、Pu 無關、純粹反映「規則本身」差異的比例關係：

| 增益 | L1 / Ku、Pu 表示式 | L2 / Ku、Pu 表示式 | **L2 ÷ L1 比例** |
|---|---|---|---|
| Kp | 0.200 Ku | 0.3125 Ku | **× 1.56** |
| Ki | 0.400 (Ku/Pu) | 0.142 (Ku/Pu) | **× 0.355** |
| Kd | 0.0667 (Ku·Pu) | 0.0481 (Ku·Pu) | **× 0.72** |

也就是說，對同一軸量到的同一組 Ku、Pu，**L2 (Tyreus-Luyben) 的比例增益會比 L1 (ZN No-Overshoot) 高出約 56%，但積分增益只剩約 35%，微分增益約剩 72%**。這三個比例就是 L1→L2 這次調校規則升級的本質：拿一部分比例響應的「快」，去換積分作用的「慢而穩」。

**舉例**(僅為說明比例關係的假設數值，並非實測結果)：假設某軸量得 Ku = 10、Pu = 0.2 秒：

| | L1 (ZN No-Overshoot) | L2 (Tyreus-Luyben) |
|---|---|---|
| Kp | 2.000 | 3.125 |
| Ki | 20.00 | 7.10 |
| Kd | 0.133 | 0.096 |

### 1.5 優劣綜合比較

| 項目 | L1 (ZN No-Overshoot) | L2 (Tyreus-Luyben) |
|---|---|---|
| 比例響應速度 | 較慢(Kp 較小) | **較快**(Kp 約 1.56 倍) |
| 積分糾錯速度 / 穩態收斂 | **較快**(Ki 約 2.8 倍) | 明顯較慢，但更不易因積分累積而過衝振盪 |
| 對陀螺儀高頻噪訊的微分放大 | 較明顯(Kd 較大) | 較溫和(Kd 約低 28%) |
| 增益邊際裕度的設計哲學 | 以「完全不過衝」為第一優先 | 在程序控制領域以「高穩定裕度」聞名，移植到飛控後同樣偏保守 |
| 理論出處的嚴謹度 | 原始 Ziegler-Nichols (1942) 之經典延伸規則，三項增益皆出自同一套理論 | Kp/Ti 出自 Tyreus & Luyben (1992) 的 PI 設計；Td 為後人移植到 PID 的經驗加值，非原始論文公式 |
| 較適合的情境 | 系統動態已大致掌握，想要更緊的追蹤與更快的干擾排除時 | 第一次對未知/未驗證的機架做自動調校，希望降低劇烈擺動或需中止保護的風險時 |
| 主要風險 | 若機架慣量、馬達響應與預期不同，較容易在積分項作用下出現過衝 | 收斂變慢，在實際飛行中對突發干擾(如風擾)的修正會感覺「鬆」、反應較慢 |

### 1.6 選用建議

兩者並非絕對的「新版優於舊版」，而是**保守度光譜上的兩個點**：

- **第一次調校、或對新機架/新馬達配置還沒有信心時**，建議從 **L2 (Tyreus-Luyben)** 開始，較不容易出現劇烈振盪需要緊急中止；
- **已經用 L2 跑過一輪、確認系統穩定後**，若覺得響應偏鬆、想要更緊的追蹤表現，可以切換回 **L1 系列中更激進的規則**(`ZN_CLASSIC_PID` 或 `ZN_SOME_OVERSHOOT`，兩者都還保留在 L1 的 enum 裡)做第二輪微調；
- 兩套規則都只是**起始值**，無論選哪一套，自動調校完成後仍建議搭配實際台架/飛行表現手動微調(尤其是 D 項，因為陀螺儀噪訊特性與機架共振頻率因板子而異，公式給的只是合理起點)。

## 2. 四份程式介紹

### 2.1 共同基礎(四份皆同)

- **硬體**：BMI088 (I2C, 0x18/0x69)、X 型四馬達混控、RP2350/RP2354A、20kHz PWM。
- **安全機制**：傾角 > 60° 自動斷電、`MOTOR_MAX_LIMIT=216`(85%)輸出上限、繼電器振幅飽和檢查。
- **姿態估算**：Mahony AHRS 互補濾波器，陀螺儀讀數(BMI088 原始輸出為 deg/s)在送入濾波器前都正確轉換為 rad/s。
- **調校量測機制**：`RelayTuneState` 狀態機、`relayStep()`、`finalizeAxisTuning()` 完全相同的繼電器回授邏輯(差異只在換算公式，見第 1 節)。
- **未解鎖時的積分保護**：四份程式目前皆已確保未解鎖(`is_armed=false`)時，PID/雙環運算(含所有積分項)完全不執行，避免停機等待期間積分漂移、解鎖瞬間輸出突變。

### 2.2 架構分類：單環 (Angle) vs 雙環串級 (Dual)

- **Angle 系列**(`L1_Angle`、`L2_Angle`)：單一個角度 PID 迴路，`output = Kp·(目標角度-量測角度) + Ki∫+ Kd·d/dt`，結構簡單、適合先驗證感測器/馬達/混控是否正常。
- **Dual 系列**(`L1_Dual`、`L2_Dual`)：外環(角度，純 P，固定 Kp=4.5)算出期望角速度，內環(角速度，完整 PID)再算出馬達修正量——這是最終飛控韌體會採用的串級架構，自動調校的目標是內環(角速度)的 Kp/Ki/Kd。

兩種架構共用同一套繼電器回授量測機制：調校時都是繼電器直接驅動最終輸出(等同把外環+內環都跳過)，量測對象實質是「PWM → 角度」這個完整受控體；Dual 系列把換算出的增益寫入內環(角速度)增益，這是架構設計上的簡化處理，效果尚可作為合理起始值，但嚴格的控制理論上仍與內環真正對應的「PWM → 角速度」受控體有差異，這點先前已經討論過，不在此文件重複展開。

### 2.3 `CURIO_L1_Angle_AutoTune.ino`

| | |
|---|---|
| 架構 | 單環角度 PID |
| 調校規則 | `ZN_NO_OVERSHOOT`(可選 4 種 ZN 規則) |
| 指令 | `t`=全自動(Roll→Pitch)、`r`=僅Roll、`p`=僅Pitch、`x`=緊急中止 |
| 解鎖方式 | 開機校正完成後**自動解鎖** |
| 中止行為 | 歸零積分項，**不斷電**，回到正常 PID 平衡 |
| 初始增益 | Roll/Pitch: Kp=1.2, Ki=0.05, Kd=0.15；Yaw: 純P，Kp=1.5 |

四份程式中最單純的一版，適合用來最先驗證測試架的感測器讀值、馬達轉向與混控方向是否正確。

### 2.4 `CURIO_L1_Dual_AutoTune.ino`

| | |
|---|---|
| 架構 | 雙環串級(外環角度P + 內環角速度PID) |
| 調校規則 | `ZN_NO_OVERSHOOT`(可選 4 種 ZN 規則) |
| 指令 | `G`=解鎖並開始全自動調校(Roll→Pitch)、`S`=中止並斷電 |
| 解鎖方式 | 需送出 `G`，解鎖與啟動調校同時發生 |
| 中止行為 | 斷電 + 歸零內環積分項 |
| 初始增益 | 外環 Kp=4.5(純P)；內環 Roll/Pitch: Kp=0.8, Ki=0.04, Kd=0.01；Yaw 內環: Kp=1.5, Ki=0.02 |

這是將 L1 的 ZN 調校規則「移植」到雙環架構後的版本，定位是：當您想用較激進、收斂較快的 ZN_NO_OVERSHOOT 規則，但又需要實際飛控所用的雙環串級結構時的選擇。

### 2.5 `CURIO_L2_Angle_AutoTune.ino`

| | |
|---|---|
| 架構 | 單環角度 PID |
| 調校規則 | `TYREUS_LUYBEN_PID`(enum 內仍保留全部 5 種規則供切換) |
| 指令 | 與 L1_Angle 完全相同：`t`/`r`/`p`/`x` |
| 解鎖方式 | 開機校正完成後自動解鎖 |
| 中止行為 | 歸零積分項，不斷電 |
| 初始增益 | 與 L1_Angle 相同(1.2/0.05/0.15、Yaw 1.5) |

與 `L1_Angle` 在架構、指令、安全機制上完全一致，唯一差異就是換算公式換成 Tyreus-Luyben(見第 1 節)。適合在 `L1_Angle` 驗證硬體沒問題後，比較兩種調校規則在單環架構下的實際響應差異。

### 2.6 `CURIO_L2_Dual_AutoTune.ino`

| | |
|---|---|
| 架構 | 雙環串級(外環角度P + 內環角速度PID) |
| 調校規則 | `TYREUS_LUYBEN_PID`(enum 內保留 `ZN_CLASSIC_PID`/`ZN_NO_OVERSHOOT`/`TYREUS_LUYBEN_PID` 三種) |
| 指令 | `G`=解鎖並開始全自動調校、`S`=中止並斷電 |
| 解鎖方式 | 需送出 `G` |
| 中止行為 | 斷電；**內環積分項目前仍不會在中止時歸零**(與 L1_Dual 的差異點，視需要可再補上) |
| 初始增益 | 與 L1_Dual 相同的外環/內環初始值(外環 Kp=4.5；內環 0.8/0.04/0.01；Yaw 1.5/0.02) |

這是目前最終飛控韌體最直接的雛形：雙環串級架構 + 偏保守的 Tyreus-Luyben 調校規則。建議的使用順序是：先用 `L2_Angle` 或 `L1_Angle` 驗證感測器與混控方向，再上 `L2_Dual` 做第一次雙環自動調校(較保守、不易劇烈擺動)，最後視響應快慢需求再考慮切到 `L1_Dual` 的 ZN 規則做第二輪微調。

## 4. 結語與後續建議

- L1 與 L2 的核心差異是**調校規則**(ZN No-Overshoot vs Tyreus-Luyben)，不是架構；四份程式剛好把「規則世代 × 架構」兩個維度交叉組合成完整矩陣，方便互相比較。
- 雙環架構(Dual)的繼電器測試對象與套用對象之間仍有理論上的不一致(測角度、套用到角速度內環增益)，這是未來若要進一步提升調校精確度時，最值得優先處理的項目。
- 目前四份程式在解鎖保護、陀螺儀單位、感測器函式呼叫上已經一致且正確；唯一尚待您決定是否補上的是 `L2_Dual` 中止調校時的內環積分項歸零(`L1_Dual` 已有、`L2_Dual` 目前沒有)。
